# go-logger: 高性能 Golang 日志库 (GitHub)

**链接**: [https://github.com/donnie4w/go-logger](https://github.com/donnie4w/go-logger)

## 1. 内容概要
这是一个 Go 语言实现的高性能日志库，主打**高并发下的极致性能**。

**核心特性**：
*   **高性能**：Benchmark 显示其在高并发场景下的延迟（~570ns/op）是标准库 `log` (~5000ns/op) 和 `zap` (~6700ns/op) 的 **1/10**。
*   **低内存**：内存分配（165 B/op）显著低于其他库。
*   **丰富的文件管理**：
    *   支持按时间（小时/天/月）分割。
    *   支持按大小分割（Rolling File）。
    *   支持自动压缩（Gzip）。
    *   支持文件数量限制（Loop）。
*   **Slog 集成**：可以作为 Go 1.21+ 标准库 `slog` 的后端。

## 2. 对 LogSentinel 项目的帮助

### 2.1 项目开发参考
*   **Benchmark 对标**：虽然 LogSentinel 是 C++ 项目，但我们在撰写论文或进行性能测试时，可以参考 `go-logger` 的测试方法（高并发写入延迟、内存分配）。如果我们未来提供 Go SDK，它是一个强劲的竞争对手/参考对象。
*   **日志轮转 (Log Rotation)**：`go-logger` 提供了非常全面的日志轮转策略（按大小、按时间、混合模式）。LogSentinel 的 `SqlitePersistence` 目前主要关注存储，但对于本地日志文件（如果有的话），应该参考这种成熟的轮转设计。
